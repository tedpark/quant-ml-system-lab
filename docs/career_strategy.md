# Career Strategy

This document explains how this repository fits into a broader transition toward financial ML, AI quant engineering, and overseas opportunities.

It is intentionally written as a public-safe strategy note. It does not claim live trading performance, private alpha, or production trading deployment.

## Core Direction

The strongest career direction is not "stock prediction with AI." It is:

```text
Production ML Engineer for financial time-series and AI quant systems
```

More specifically:

```text
Senior software engineer moving into production ML engineering,
with a focus on financial data pipelines, model training, walk-forward evaluation,
risk-aware decisioning, inference serving, and operational monitoring.
```

This positioning uses existing software engineering experience instead of discarding it.

## Target Roles

Best-fit roles:

- Financial ML Engineer
- AI Quant Developer
- Production Quant ML Engineer
- ML Engineer, Trading Systems
- Quant Developer with ML focus
- Research Engineer in fintech
- MLOps / Inference Engineer for model systems

Secondary roles:

- LLM / RAG Evaluation Engineer
- ML Platform Engineer
- AI Application Engineer with strong evaluation and serving responsibilities

Less direct fit without additional credentials:

- pure Quant Researcher
- Deep Learning Research Scientist
- foundation-model research roles

Those roles usually require stronger academic signals, publications, or large-scale model training experience.

## Why This Repo Exists

This repository is meant to offset the lack of a formal ML job title by creating verifiable evidence:

- reproducible code
- tests
- validation discipline
- benchmark reports
- serving contracts
- monitoring reports
- public writing anchors
- clear public/private boundary

The repo should prove:

```text
I can understand models, train them, evaluate them honestly,
serve them through an API, and connect them to operational metrics.
```

## Two-Track Strategy

The recommended approach is a two-track strategy:

```text
Track A: portfolio + job search
Track B: part-time graduate study preparation
```

The important rule:

```text
Do not pause the portfolio and job search just to wait for graduate school.
```

Graduate school can strengthen the signal, especially overseas, but the fastest hiring evidence still comes from public execution.

## Track A: Portfolio and Job Search

Primary artifacts:

1. executable GitHub repository
2. benchmark reports
3. model serving / demo layer
4. technical writing series
5. resume / portfolio / book landing-page material

The execution details are tracked in:

- `docs/portfolio_artifacts.md`
- `docs/ai_quant_strategy.md`
- `docs/recruiting_notes.md`

Weekly operating cadence:

- improve one repo artifact
- publish or draft one technical section
- apply to overseas / remote roles
- practice one ML system design or quant validation story

## Track B: Part-Time Graduate Study

Part-time graduate study can help, especially for international credibility.

The recommended option is:

```text
Georgia Tech OMSCS, Machine Learning specialization
```

### Georgia Tech OMSCS

Why it fits:

- strong global CS brand
- online format can be combined with work
- degree is a real MS in Computer Science
- Machine Learning specialization aligns with the project
- useful signal for overseas recruiters and hiring managers

Important limitations:

- OMSCS is online and does not provide a US F-1 / OPT path.
- It helps the credential signal, but it does not directly solve US work authorization.
- It should be run in parallel with public portfolio work.
- A one-course-per-semester pace is more realistic while working.

Recommended course direction:

- Machine Learning
- Reinforcement Learning
- Machine Learning for Trading, if available and appropriate
- Deep Learning
- Graduate Algorithms
- Data / analytics / ML systems courses

### Georgia Tech OMSA

OMSA is a reasonable secondary option for analytics and data-science roles, but OMSCS is the better fit for:

- ML Engineer
- AI Quant Developer
- Production ML Engineer
- software-engineering-heavy ML roles

### Domestic Part-Time AI Graduate Programs

These can be useful if they provide:

- credible research supervision
- publishable project work
- industry network
- strong thesis or paper output

They are less useful if they only add coursework without visible artifacts.

## Overseas Strategy

### United States

The US is high value but difficult from outside the country.

Realistic routes:

1. remote or contractor role with a US company
2. global company in Korea followed by internal transfer
3. on-campus US graduate program with F-1 / OPT route
4. long-term O-1 / NIW path if public evidence becomes strong enough

OMSCS helps with credibility, but it does not provide OPT.

Near-term US focus:

```text
US remote, global teams, contractor roles, and companies open to distributed engineering.
```

### Canada

Canada can be more practical than the US for tech immigration and employer-sponsored paths.

Target roles:

- ML Engineer
- Financial ML Engineer
- Data / ML Platform Engineer
- fintech engineering roles

Strategy:

- apply to Canadian AI, fintech, data platform, and trading-adjacent companies
- position as senior software + production ML
- negotiate remote-first or relocation after trust is established

### United Kingdom

The UK can be attractive for fintech and AI roles.

Longer-term angle:

- Global Talent-style evidence may become possible if there is enough public proof:
  - English technical book
  - public repo
  - technical writing
  - open-source traction
  - conference or podcast appearances
  - strong recommendation letters

### Singapore / EU / Japan

These should be considered alongside the US:

- Singapore: fintech, quant, trading systems, AI engineering
- Germany / Netherlands: English-speaking engineering roles
- Japan: relatively accessible regional market with fintech and AI roles

## Final Recommendation

The strongest strategy is:

```text
Build public proof first.
Prepare OMSCS in parallel.
Use the English book and repo as the international credibility layer.
Target production ML / AI quant roles before pure research roles.
```

The goal is not to look like a junior ML candidate. The goal is to look like a senior engineer who can build production-grade financial ML systems.
